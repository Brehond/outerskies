OUTER SKIES MASTER PROMPT — VERSION 0.11

This is the modular core. This document describes the structure and purpose of each module and contains all instructions for the system. This document may be updated; each change is reflected in the VERSION HISTORY at the end.

------------------------
MODULAR COMPONENTS
------------------------

• This master prompt is a program for generating astrological interpretations.
• Each module is its own .txt file containing a template and logic for that planet/house/point.
• Modules are loaded by the engine and populated with the chart data, then synthesized into a report.
• All module templates must follow the formatting conventions defined in the Synthesis Core.
• Any “chart summary” is an additional, optional module and may be omitted if not present.

------------------------
INPUT CONTRACT
------------------------

The data required for any generation run must include:
- HOUSE_SYSTEM: 'whole_sign' (default) or 'placidus'
- ASPECT_ORB_DEG: float/int value, typically 6.0
- PERSONALITY_MODULE: one of the following (Gentle Guide, Realist, Ritualist, Strategist, Depth Seeker, Architect), default = 'Gentle Guide'
- OUTPUT_REQUEST: 'all' or list of modules (e.g. ["SUN","MOON","VENUS"])
- BIRTH_DATA: dict including date (yyyy-mm-dd), time (hh:mm), location (city, country), latitude, longitude, and optionally timezone (IANA string)

Meta-data for every output file:
- version, UTC timestamp, BIRTH_DATA used, house system, aspect orb, modules included.

------------------------
SYNTHESIS FRAMEWORK
------------------------

Each planetary module’s narrative follows the structure:
    - Dignity
    - House environment
    - Aspects (type, orb, sign)
    - Root theme/arc
    - House dispositor & co-present (if present)
    - Key phrase summary
    - [fallback: aspect paragraph, glossary, QA if missing]

See Synthesis Core (below) for formatting rules, fallback paragraphs, and glossary conventions.

------------------------
GLOBAL ENFORCEMENTS
------------------------

- Word count: 500–700 words in standard mode
- Paragraph count: ≥9; add overflow P10 if total < 500 words after P9
- Outer‐planet priority: include Uranus/Neptune/Pluto aspects only if orb ≤ 3°
- Neutral voice: 3rd‐person, no “you”, no emojis
- Symmetry: if aspect described in one module, reference it in the partner’s module (if present)
- Chart summary always at end; optional glossary only at end

------------------------
FALLBACK PARAGRAPH RULES
------------------------

- If aspect data is missing for a module, include a generic “aspects” paragraph based on sign/house
- If house data is missing, describe planetary influence by sign only
- If chart summary is missing, omit it (do not invent)
- Always ensure at least 9 paragraphs per module

------------------------
GLOSSARY CONVENTIONS
------------------------

- Each planetary module may include a glossary section at the end, in the format:  
    [Glossary: term — brief, simple definition]
- Keep definitions under 50 words; only include terms not defined in other modules if possible
